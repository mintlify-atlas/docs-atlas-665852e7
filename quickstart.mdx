## Welcome to SaludYa API

This guide provides example API requests for the SaludYa medical appointment management system. You'll see how authentication works, how to retrieve patient information, and create appointments.

## API Base URL

The API is designed to be hosted at:
```
https://api.saludya.com/v1
```

## Prerequisites

When implementing this API, clients will need:
- An API key for authentication
- A tool to make HTTP requests (curl, Postman, or any HTTP client)

<Steps>
  <Step title="API Key Authentication">
    The API uses bearer token authentication. API keys should be included in the `Authorization` header.
    
    <Warning>
      Keep your API key secure and never commit it to version control. Treat it like a password.
    </Warning>
  </Step>

  <Step title="Authenticate Your Request">
    All API requests require authentication using your API key in the `Authorization` header with the `Bearer` scheme.

    <CodeGroup>
    ```bash curl
    curl https://api.saludya.com/v1/auth/verify \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```

    ```javascript JavaScript
    const response = await fetch('https://api.saludya.com/v1/auth/verify', {
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY'
      }
    });
    const data = await response.json();
    console.log(data);
    ```

    ```python Python
    import requests

    headers = {
        'Authorization': 'Bearer YOUR_API_KEY'
    }

    response = requests.get(
        'https://api.saludya.com/v1/auth/verify',
        headers=headers
    )
    print(response.json())
    ```
    </CodeGroup>

    **Response:**
    ```json
    {
      "status": "success",
      "data": {
        "organization_id": "org_7Yh8KpQmN3",
        "organization_name": "City Medical Center",
        "environment": "production"
      }
    }
    ```
  </Step>

  <Step title="Retrieve Available Doctors">
    Before creating an appointment, let's fetch the list of available doctors in your organization.

    <CodeGroup>
    ```bash curl
    curl https://api.saludya.com/v1/doctors?specialty=cardiology \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```

    ```javascript JavaScript
    const response = await fetch(
      'https://api.saludya.com/v1/doctors?specialty=cardiology',
      {
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY'
        }
      }
    );
    const doctors = await response.json();
    console.log(doctors);
    ```

    ```python Python
    import requests

    headers = {
        'Authorization': 'Bearer YOUR_API_KEY'
    }

    params = {
        'specialty': 'cardiology'
    }

    response = requests.get(
        'https://api.saludya.com/v1/doctors',
        headers=headers,
        params=params
    )
    print(response.json())
    ```
    </CodeGroup>

  <Step title="Get Doctor Availability">
    Check available time slots for a specific doctor on a given date.

    <CodeGroup>
    ```bash curl
    curl "https://api.saludya.com/v1/doctors/doc_9KmLp3xQw2/availability?date=2026-03-10" \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```

    ```javascript JavaScript
    const doctorId = 'doc_9KmLp3xQw2';
    const date = '2026-03-10';

    const response = await fetch(
      `https://api.saludya.com/v1/doctors/${doctorId}/availability?date=${date}`,
      {
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY'
        }
      }
    );
    const availability = await response.json();
    console.log(availability);
    ```

    ```python Python
    import requests

    headers = {
        'Authorization': 'Bearer YOUR_API_KEY'
    }

    doctor_id = 'doc_9KmLp3xQw2'
    params = {

    }

    response = requests.get(
        f'https://api.saludya.com/v1/doctors/{doctor_id}/availability',
        headers=headers,
        params=params

    print(response.json())
    ```
    </CodeGroup>

  <Step title="Create an Appointment">
    Now let's create a medical appointment for a patient.

    <CodeGroup>
    ```bash curl
    curl -X POST https://api.saludya.com/v1/appointments \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "patient": {
          "id": "pat_4XmNp9kLq7",
          "name": "Juan Perez",
          "email": "juan.perez@example.com",
          "phone": "+52-555-1234"
        },
        "doctor_id": "doc_9KmLp3xQw2",
        "date": "2026-03-10",
        "start_time": "09:00",
        "end_time": "09:30",
        "reason": "Cardiac consultation",
        "type": "in-person"
      }'
    ```

    ```javascript JavaScript
    const appointmentData = {
      patient: {
        id: 'pat_4XmNp9kLq7',
        name: 'Juan Perez',
        email: 'juan.perez@example.com',
        phone: '+52-555-1234'
      },
      doctor_id: 'doc_9KmLp3xQw2',
      date: '2026-03-10',
      start_time: '09:00',
      end_time: '09:30',
      reason: 'Cardiac consultation',
      type: 'in-person'
    };

    const response = await fetch('https://api.saludya.com/v1/appointments', {
      method: 'POST',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(appointmentData)
    });

    const appointment = await response.json();
    console.log(appointment);
    ```

    ```python Python
    import requests

    headers = {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    }

    appointment_data = {
        'patient': {
            'id': 'pat_4XmNp9kLq7',
            'name': 'Juan Perez',
            'email': 'juan.perez@example.com',
            'phone': '+52-555-1234'
        },
        'doctor_id': 'doc_9KmLp3xQw2',
        'date': '2026-03-10',
        'start_time': '09:00',
        'end_time': '09:30',
        'reason': 'Cardiac consultation',
        'type': 'in-person'
    }

    response = requests.post(
        'https://api.saludya.com/v1/appointments',
        headers=headers,
        json=appointment_data
    )
    print(response.json())
    ```

## Common Use Cases

<AccordionGroup>
  <Accordion title="Canceling an Appointment">
    Use the `PATCH` endpoint to update appointment status:
    ```bash
    curl -X PATCH https://api.saludya.com/v1/appointments/apt_8QrLm5xTw3 \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"status": "cancelled", "cancellation_reason": "Patient request"}'
    ```
  </Accordion>

  <Accordion title="Searching Appointments by Patient">
    Filter appointments by patient ID:
    ```bash
    curl "https://api.saludya.com/v1/appointments?patient_id=pat_4XmNp9kLq7" \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```
  </Accordion>

  <Accordion title="Creating a New Patient">
    Register a new patient before creating their first appointment:
    ```bash
    curl -X POST https://api.saludya.com/v1/patients \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Maria Garcia",
        "email": "maria.garcia@example.com",
        "phone": "+52-555-5678",
        "date_of_birth": "1985-05-15",
        "insurance_provider": "IMSS"
      }'
    ```
  </Accordion>
</AccordionGroup>

## Rate Limits

The SaludYa API implements rate limiting to ensure system stability:

- **Free tier**: 100 requests per minute
- **Pro tier**: 1,000 requests per minute
- **Enterprise tier**: Custom limits

Rate limit information is included in response headers:
```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1709740800
```
