---
title: 'Create Doctor'
api: 'POST /api/doctors'
description: 'Add a new doctor to the system'
---

## Request Body

<ParamField body="name" type="string" required>
  Full name of the doctor (e.g., "Dr. Maria Garcia")
</ParamField>

<ParamField body="specialty" type="string" required>
  Medical specialty or area of expertise (e.g., "Cardiology", "Pediatrics", "Dermatology")
</ParamField>

<ParamField body="license_number" type="string" required>
  Professional medical license number. Must be unique in the system.
</ParamField>

<ParamField body="email" type="string" required>
  Doctor's professional email address. Must be a valid email format.
</ParamField>

<ParamField body="phone" type="string" required>
  Contact phone number with country code (e.g., "+1-555-0123")
</ParamField>

<ParamField body="bio" type="string">
  Professional biography, credentials, and areas of expertise. Maximum 1000 characters.
</ParamField>

<ParamField body="available" type="boolean" default="true">
  Whether the doctor is accepting new patients
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique identifier for the newly created doctor
</ResponseField>

<ResponseField name="name" type="string">
  Full name of the doctor
</ResponseField>

<ResponseField name="specialty" type="string">
  Medical specialty
</ResponseField>

<ResponseField name="license_number" type="string">
  Professional medical license number
</ResponseField>

<ResponseField name="email" type="string">
  Doctor's contact email address
</ResponseField>

<ResponseField name="phone" type="string">
  Doctor's contact phone number
</ResponseField>

<ResponseField name="bio" type="string">
  Professional biography
</ResponseField>

<ResponseField name="available" type="boolean">
  Availability status
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of creation
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of last update
</ResponseField>

## Next Steps

2. **View doctor details** - Retrieve the doctor's information using [Get Doctor](/api/doctors/get)
3. **Enable appointments** - Once schedules are configured, patients can book appointments with this doctor
